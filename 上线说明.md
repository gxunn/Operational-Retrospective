# Ubuntu 服务器上线说明

## 上线前需要准备

1. 一台 Ubuntu 24.04 服务器，开放 22、80、443 端口。
2. 一个域名，并把域名的 A 记录指向服务器公网 IP。
3. 服务器安装 Docker Engine 和 Docker Compose 插件。

## 上传后的启动步骤

在服务器项目目录中复制生产配置示例：

```bash
cp .env.production.example .env.production
```

编辑 `.env.production`，至少填写域名、管理员强密码、会话密钥和 OpenAI 密钥。所有密钥只保存在服务器文件中，不要发到聊天或代码仓库。

启动：

```bash
DOMAIN=你的域名 docker compose -f compose.production.yml up -d --build
```

检查：

```bash
docker compose -f compose.production.yml ps
docker compose -f compose.production.yml logs --tail=100 app caddy
```

Caddy 会自动申请 HTTPS 证书。成功后访问 `https://你的域名`。

## 更新

上传新版代码后执行：

```bash
DOMAIN=你的域名 docker compose -f compose.production.yml up -d --build
```

## 备份

需要备份 `data`、`uploads`、`reports` 三个目录。复制数据库前先暂停应用容器，复制完成后立即恢复：

```bash
docker compose -f compose.production.yml pause app
```

完成文件复制后：

```bash
docker compose -f compose.production.yml unpause app
```

不要使用带删除数据卷参数的命令。
